We are making a web based game.

The goal is to make a game that is essentially a copy of the quake mode defrag.

We are going to start with an MVP with the following:

- Our main focus should be the 3D engine and the gameplay feel
- We don't need multiplayer but we need a "ghost" system, i.e. it's easy to race against past performances.
- We want to keep costs to virtually $0 so we're not going to have a central server with player accounts and whatnot.
  We want the main website to be static, meaning it hosts assets and clients play them, but that's that
- This means they save their ghost/replay file locally, and are able to share them

For the MVP we are going to completely ignore implementing the ghost system, but in the future we will want to implement ghosts and player accounts
with a central server where player can save their times and ghosts and share them with the community.
